# __Meeting Manager:__

<br>

## 🎯 __Target__
__This is an academic project! The challenge is:__
* Build a REST API with CRUD operations.
* Entities with at least 1..N and N..N relationships.
* Any framework can be used.
* All concepts of OOP need to be applied.

## 📓 __Project description__
__Management meeting reservation system and the scope of this project is:__
* User: Contains an id, name, and email.
* Meeting Creation: A user can create zero or multiple meetings; each meeting is associated with exactly one creator.
* Meeting Participation: A meeting must have one or more participants. A user can participate in one or more meetings.
* Meeting Management: Users can create, update, delete, and list meetings.
* Meeting Details: Every meeting includes an id, title, start_time, and end_time. Meetings are categorized as either virtual or presential:
   - Virtual Meeting: Includes a meeting access link.
   - Presential Meeting: Requires a room assignment.
* Room: Contains an id, name, and capacity. A room can be scheduled for zero or multiple presential meetings, and each presential meeting must be associated with exactly one room.
